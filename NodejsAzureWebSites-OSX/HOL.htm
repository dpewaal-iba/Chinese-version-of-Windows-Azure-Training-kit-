使用 Windows Azure 网站构建和发布 Node.js 应用程序 (OS X)

概述

在此动手实验中，您将学习针对 Node.js 应用程序的 Windows Azure 网站服务的基础知识。在练习 1 中，您将看到如何使用 Windows Azure 门户创建一个网站，然后使用 GIT 发布一个“Hello World”Node.js 应用程序。在练习 2 中，您将学习如何使用 Windows Azure 命令行工具发布应用程序。

目标

在此动手实验中，您将学习如何使用以下各项创建和发布 Node.js 应用程序：

  • Git
  • 适用于 Mac 和 Linux 的 Windows Azure 命令行工具

先决条件

下面是完成此动手实验需要满足的先决条件：

  • Git 版本控制系统
  • Node.js

  • 适用于 Mac 和 Linux 的 Windows Azure 命令行工具（仅用于练习 2）

    注意：如果您没有安装适用于 Mac 和 Linux 的 Windows Azure 命令行工具，请打开一个终端，然后运行以下命令：

    ShellScript
    sudo npm install azure-cli -g

    通过使用 -g，Windows Azure 命令行工具将全局安装在您的计算机上。这意味着，您将可以从任何位置执行 Azure 命令。

  • 启用了网站预览的 Windows Azure 订阅 - 注册以免费试用

注意：本实验设计为使用 OS X 操作系统。

练习

此动手实验包括以下练习：

  1. 使用 GIT 构建和发布 Node.js 网站
  2. 使用命令行工具构建和发布 Node.js 网站

练习 1：使用 GIT 构建和发布 Node.js 网站

在第一个练习中，您将在 Windows Azure 门户中创建一个新网站，创建一个“Hello World”Node.js 应用程序并最后发布此应用程序（利用 Windows Azure 提供的新的 GIT 发布功能）。

任务 1 - 创建在 Windows Azure 中托管的新网站

  1. 转到 Windows Azure 门户，使用与您的订阅关联的 Microsoft 帐户凭据登录。

    登录到 Windows Azure 门户

  2. 在 Windows Azure 门户主页中，单击位于底部栏中的“新建”按钮。

    创建新网站

  3. 单击“网站”|“快速创建”。为这个新网站提供一个可用 URL，选择某一区域，然后单击“创建网站”。

    使用“快速创建”选项创建新网站

  4. 等待新网站创建完成。

    创建新网站状态

  5. 创建了网站后，单击 URL 链接以便检查这个新网站是否工作正常。

    浏览到新网站

    运行中的网站

  6. 回到门户，然后单击网站名称列以转到网站的仪表板。

    选择“仪表板”选项卡

  7. 单击“设置 Git 发布”链接。

    设置 Git 发布

  8. 等待几秒钟，直到 Git 存储库就绪。

    Git 存储库已创建

  9. 不要关闭该门户。

任务 2 - 创建新的 Node.js 网站

  1. 在可以轻松访问的位置中创建一个新文件夹。

  2. 打开您喜爱的文本编辑器并复制以下代码，该代码将字符串 Hello World 发送到浏览器。使用 server.js 名称将该文件保存到之前创建的文件夹中。

    Javascript
    var http = require('http');
var port = process.env.port || 1337; http.createServer(function (req, res) { res.writeHead(200, { 'Content-Type': 'text/plain' }); res.end('Hello World\n'); }).listen(port);

  3. 创建一个新的文本文件并且粘贴以下代码。将该文件以 package.json 的名称另存到同一个文件夹中。

    Json
    { "name": "HelloWorld", "version": "0.0.1", "description": "", "main": "./server.js", "engines": { "node": ">= 0.6.0" } }

    注意：package.json 文件指示 Node.js 包管理器 (npm) 如何建立包结构以及哪些是模块依赖项。因为此应用程序非常简单，所以它不要求依赖项，但在更复杂的应用程序中，您将可能需要指定 "dependencies" 属性。在使用 Git 发布应用程序时，Windows Azure 平台将自动安装在此文件中声明的所有依赖项。

  4. 在文本编辑器中，创建一个新的文件并且粘贴以下代码。此配置指示 server.js 文件是应该由 iisnode 模块处理的 Node.js 应用程序。

    注意：iisnode 项目提供一个本机 IIS 7.x 模块，该模块将允许在 IIS 7.x 中承载 node.js 应用程序。

    XML
    <?xml version="1.0" encoding="utf-8"?>
<configuration>
    <system.webServer>
          <modules runAllManagedModulesForAllRequests="false" />

          <!-- indicates that the server.js file is a node.js application to be handled by the iisnode module -->
          <handlers>
            <add name="iisnode" path="server.js" verb="*" modules="iisnode" />
          </handlers>
          <rewrite>
            <rules>
              <clear />
              <rule name="app" enabled="true" patternSyntax="ECMAScript" stopProcessing="true">
                <match url="server\.js.+" negate="true" />
                <conditions logicalGrouping="MatchAll" trackAllCaptures="false" />
                <action type="Rewrite" url="server.js" />
              </rule>
            </rules>
          </rewrite>
    </system.webServer>
</configuration>

  5. 将该文件以 web.config 的名称另存到您的网站文件夹中。

任务 3 - 使用 Git 发布 Node.js 应用程序

  1. 返回到门户。打开网站的“仪表板”，复制该页面底部的 Git Clone Url 值。

    复制 Git Clone Url

  2. 为了发布您的本地文件，请打开“终端”控制台，将目录更改到您的应用程序文件夹并运行以下命令。

    终端
    git init git add . git commit -m "initial commit"

    注意：您可以在 http://git-scm.com/documentation 中了解与 Git 命令有关的详细信息。

  3. 若要添加远程 Windows Azure 存储库并推送文件，请运行以下命令。使用从门户获取的值替换 {Git Clone Url} 占位符。

    终端
    git remote add azure {Git Clone Url} git push azure master

  4. 出现提示时，请输入部署凭据。

    注意：部署凭据不同于与您的 Windows Azure 订阅关联的 Microsoft 帐户，并适用于与您的订阅关联的所有 Windows Azure 网站。如果您不知道自己的部署凭据，使用管理门户可以轻松重置它们。打开网站的“仪表板”页，单击“重置部署凭据”链接。提供一个新密码，然后单击“确定”。

    重置部署凭据

  5. 转到网站的“仪表板”，单击“站点 URL”链接以便确保该网站正在运行。将会显示 Hello Word 消息。

    运行 Node.js 应用程序

练习 2：使用命令行工具构建和发布 Node.js 网站

在此练习中，您将学习如何使用适用于 Mac 和 Linux 的 Windows Azure 命令行工具管理网站。

注意：以下步骤假定您已安装了适用于 Mac 和 Linux 的 Windows Azure 命令行工具。

  1. 打开“终端”控制台并且执行以下命令以便下载您的帐户的“发布设置”文件。此时将弹出一个新的浏览器窗口。使用与您的 Windows Azure 订阅相关联的 Microsoft 帐户凭据进行登录。

    终端
    azure account download

    您的发布设置将下载到浏览器的默认下载文件夹（通常为 ~/Downloads），并将显示一个新页面，其中提供了相关的信息。

    下载发布设置文件

  2. 返回到“终端”控制台，通过运行以下命令导入在上一个步骤中下载的文件，并且在 {publish_settings_file} 占位符中指定“发布设置”文件位置。

    终端
    azure account import {publish_settings_file}

  3. 执行练习 1 中的任务 2 以便创建一个 Node.js 网站，但将文件放置于其他文件夹中。

  4. 返回到“终端”控制台，更改目录以便转到放置了文件的文件夹。

  5. 在启用了“GIT 发布”功能的情况下，运行以下命令以创建 Windows Azure 网站。

    终端
    azure site create --git

    在系统提示时提供一个网站名称，例如 nodejsappOSXcli。然后，选择一个区域并继续。

    创建新网站并启用 GIT 发布

    注意：在运行命令 site create 时通过指定 --git，您无需运行附加的命令来初始化 Git 存储库或添加远程 Git，因为这两个任务将由 Windows Azure 命令行工具自动完成。

  6. 现在，您将应用程序文件添加到 Git 存储库并且将它们推送到 Windows Azure 网站。为此，返回到命令提示符并且执行以下命令。出现提示时，提供您的部署凭据。

    终端
    git add . git commit -m "initial commit" git push azure master

    推送网站文件

    注意：部署凭据不同于与您的 Windows Azure 订阅关联的 Microsoft 帐户，并适用于与您的订阅关联的所有 Windows Azure 网站。如果您不知道自己的部署凭据，使用管理门户可以轻松重置它们。打开网站的“仪表板”页，单击“重置部署凭据”链接。提供一个新密码，然后单击“确定”。

    重置部署凭据

  7. 运行以下命令以便在 Windows Azure 门户中打开您的网站的已发布部署。

    终端
    azure site portal

    在 Windows Azure 门户中，从列表中选择您的网站，然后打开网站的“部署”页。签出最新的部署。

    网站部署

  8. 执行以下命令以便浏览到已发布网站并且确保其正常工作。

    终端
    azure site browse

    网站正常工作
