设置 Windows Azure 虚拟机

概述

在此实验中，您将使用 Web 浏览器中的 Windows Azure 管理控制台创建一个新的虚拟网络，然后从库映像创建一个新的 Windows Server 2012 虚拟机。创建虚拟网络不是从库映像创建新的虚拟机所必需的，但控制分配给虚拟机的 IP 地址或使 VPN 连接能够返回到公司内部网络需要创建虚拟网络。

注意：如果您是第二次按照此 HOL 执行，则可以跳过练习 1 并且移到练习 2，以便在您已创建的现有虚拟网络上创建附加虚拟机。

目标

在此动手实验中，您将学习如何执行以下操作：

  • 配置新的虚拟网络
  • 从库映像部署新的虚拟机

先决条件

下面是完成此动手实验需要满足的先决条件：

  • Windows Azure 订阅 - 注册以免费试用。

注意：要运行完整的动手实验，您必须要有网络连接。

练习

此动手实验包括以下练习：

  1. 创建新的虚拟网络
  2. 从库映像创建新的 Windows Server 2012 虚拟机

完成此实验的估计时间：30 分钟

练习 1：创建新的虚拟网络

许多虚拟机上的工作负荷都要求永久性的 IP 地址和自提供的 DNS 名称解析。Windows Azure 中的默认内部 DNS 服务 (iDNS) 通常不是可接受的解决方案，因为分配给每个虚拟机的 IP 地址都不是永久性的。对于此解决方案，您将定义一个虚拟网络，在这里，您可以将虚拟机分配给特定子网。

此实验所用的网络配置定义的内容如下：

  • 名为 domainvnet 的虚拟网络，其地址前缀：10.0.0.0/16
  • 名为 AppSubnet 的子网，其地址前缀：10.0.10.0/24
  • 名为 DC01 的本地 DNS 服务器，其 IP 地址：10.0.10.4

练习 1 包含 1 个任务：

  1. 新建虚拟网络

任务 1 - 新建虚拟网络

第一步是使用一个新的地缘组在您的订阅中新建一个虚拟网络。

  1. 打开浏览器并转到 https://manage.windowsazure.com/。提示时，使用您的 Windows Azure 凭据登录。在 Windows Azure 门户中，单击“新建”，选择“网络”|“虚拟网络”，然后单击“自定义创建”。

    自定义创建虚拟网络

  2. 在“虚拟网络详细信息”页上的“名称”文本框中，键入 domainvnet。从“地缘组”下拉列表中选择“创建新的地缘组”选项。将该地缘组命名为 adag，然后选择最靠近您的区域。单击箭头按钮以继续。

    新建虚拟网络

  3. 在右上角，选择 CIDR 选项。然后，将地址空间起始 IP 值设置为 10.0.0.0，将其地址 CIDR 设置为 /16。最后，添加名为 AppSubnet 的一个子网，其起始 IP 为 10.0.10.0，地址 CIDR 为 /24。单击箭头按钮以继续执行后续步骤。

    添加地址空间和子网

  4. 在“DNS 服务器和本地网络”页上的“DNS 服务器”下，选择“指定新的 DNS 服务器”，然后在“名称”文本框中键入 DC01。在“IP 地址”文本框中，键入 10.0.10.4。单击加号以便定义 DNS 服务器地址。单击复选标记图标以继续。> 注意：此设置指定 Azure 将 10.0.10.4 作为 DNS 服务器分配给此子网中的虚拟机。

    创建虚拟网络

练习 2：从库映像创建新的虚拟机

您现在将从名为 DC01 的 Windows Server 2012 库映像创建新的虚拟机。如果您是首次后继续执行此练习，则应将虚拟机命名为 DC01 以外的名称，并且还要确保在创建虚拟机时连接到现有的虚拟机网络。该 DC01 虚拟机将在不同练习中使用。

练习 2 包含 1 个任务：

  1. 新建虚拟机

任务 1 - 新建虚拟机

  1. 在 Windows Azure 门户中，选择“虚拟机”部分。

    选择“虚拟机”部分

  2. 在工具栏中，单击“新建”图标。在“新建”对话框窗口中，选择“虚拟机”，然后单击“从库中”。

    从库中创建虚拟机

  3. 在“创建虚拟机”对话框窗口中，选择“平台映像”，选择 Windows Server 2012 Datacenter 映像，然后单击右箭头图标以便继续。

    选择平台映像

  4. 在“虚拟机名称”文本框中，键入 DC01。

注意：如果您是在首次后执行此练习以便创建附加虚拟机，则指定 DC01 以外的其他名称。

  1. 在“新密码”和“确认”文本框中，键入 Passw0rd!。

  2. 在“大小”下拉框中，确保选择了“小”，然后单击右箭头图标以便继续。

    创建虚拟机

  3. DNS 名称需要是唯一的，最好有一定的含义。您可以选择某一现有存储帐户，或者自动生成一个新的存储帐户。从下拉列表“区域/地缘组/虚拟网络”中选择 domainvnet 虚拟网络。单击箭头继续。

    注意：如果您是在首次后执行此练习以便创建附加虚拟机，则请确保选中选项“连接到现有虚拟机”。

    为虚拟机选择虚拟网络

  4. 在“虚拟机选项”页上，单击复选标记图标以便完成该向导。正在创建新虚拟机。

    完成虚拟机创建过程

小结

通过完成此动手实验，您学习了如何：

  • 配置新的虚拟网络
  • 从库映像部署新的虚拟机
