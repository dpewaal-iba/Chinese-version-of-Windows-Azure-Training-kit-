在 Windows Azure 中部署 Active Directory

概述

在此动手实验中，您将演练在云中部署 Active Directory 域并将新虚拟机设置到域中所需的步骤。

目标

在此动手实验中，您将学习如何执行以下操作：

  • 配置虚拟网络
  • 部署域控制器
  • 在域中创建新虚拟机

先决条件

  • Windows Azure PowerShell Cmdlet
  • 启用了虚拟机预览的 Windows Azure 订阅 - 注册以免费试用

设置

此实验需要 Windows Azure PowerShell Cmdlet。如果尚未配置它们，请参阅使用 PowerShell 实现 Windows Azure 自动化模块中的自动执行虚拟机管理动手实验。

注意：要运行完整的动手实验，您必须要有网络连接。

练习

此动手实验包括以下练习：

  1. 配置虚拟网络
  2. 部署第一个域控制器
  3. 将新虚拟机设置到域中

完成此实验的估计时间：60 分钟。

入门：获取订阅凭据

为了完成此实验，您将需要获得订阅的安全凭据。Windows Azure 允许您下载发布设置文件，其中包含在开发环境中管理帐户所需的所有信息。

任务 1 - 下载和导入发布设置文件

注意：如果您已经在之前的实验中在同一计算机上完成了这些步骤，则可以转到练习 1。

在本任务中，您将登录到 Windows Azure 门户并下载发布设置文件。此文件包含安全凭据以及有关在开发环境中使用的 Windows Azure 订阅的其他信息。然后，您将使用 Windows Azure Cmdlet 导入此文件，以便安装证书并获取帐户信息。

  1. 打开 Internet Explorer 浏览器，转到 https://windows.azure.com/download/publishprofile.aspx。

  2. 使用与您的 Windows Azure 帐户关联的 Microsoft 帐户登录。

  3. 将发布设置文件保存到本地计算机。

    下载发布设置文件

    注意：下载页面说明如何使用 Visual Studio 发布框导入发布设置文件。此实验将说明如何改用 Windows Azure PowerShell Cmdlet 执行导入。

  4. 在 Windows Azure 下的开始菜单中，右键单击 Windows Azure PowerShell，然后选择“以管理员身份运行”。

  5. 将 PowerShell 执行策略更改为 RemoteSigned。在要求确认时，按 Y，然后按 Enter。

    PowerShell
    Set-ExecutionPolicy RemoteSigned

    注意：通过 Set-ExecutionPolicy cmdlet 可以确定允许在计算机上运行的 Windows PowerShell 脚本（如果有）。Windows PowerShell 有四种不同的执行策略：

    • Restricted - 任何脚本都不能运行。Windows PowerShell 只能在交互模式中使用。
    • AllSigned - 只有由受信任的发布者签名的脚本可以运行。
    • RemoteSigned - 下载的脚本必须由受信任的发布者签名，才可以运行。
    • Unrestricted - 没有限制，所有 Windows PowerShell 脚本都可以运行。

    有关执行策略的详细信息，请参阅 TechNet 文章：http://technet.microsoft.com/zh-CN/library/ee176961.aspx

  6. 下面的脚本导入发布设置文件并使用您的帐户信息生成一个 XML 文件。您将在实验过程中使用这些值来管理您的 Windows Azure 订阅。将占位符替换为发布设置文件的路径并执行脚本。

    PowerShell
    Import-AzurePublishSettingsFile '[YOUR-PUBLISH-SETTINGS-PATH]'

  7. 执行以下命令，记录订阅名称和存储帐户名称以备本练习使用。

    PowerShell
    Get-AzureSubscription | select SubscriptionName
Get-AzureStorageAccount | select StorageAccountName

  8. 如果您使用上面的命令没有返回存储帐户，应先创建一个。

    1. 运行以下命令，确定要在其中创建存储帐户的数据中心。确保选择支持 PersistentVMRole 的数据中心。

      PowerShell
      Get-AzureLocation

    2. 创建存储帐户：

      PowerShell
      New-AzureStorageAccount -StorageAccountName '[YOUR-SUBSCRIPTION-NAME]' -Location '[DC-LOCATION]'

  9. 执行以下命令以便设置您的订阅的当前存储帐户。

    PowerShell
    Set-AzureSubscription -SubscriptionName '[YOUR-SUBSCRIPTION-NAME]' -CurrentStorageAccount '[YOUR-STORAGE-ACCOUNT]'

练习 1：配置虚拟网络

运行 Active Directory 域需要永久性 IP 地址，并需要 Active Directory 域的客户端指向启用了 AD 的 DNS 服务器。Windows Azure 中的默认内部 DNS 服务 (iDNS) 不是可接受的解决方案，因为分配给每个虚拟机的 IP 地址都不是永久性的。对于此解决方案，您将定义一个虚拟网络，在这里，您可以将虚拟机分配给特定子网。

此实验所用的网络配置定义的内容如下：

  • 名为 ADVNET 的虚拟网络，其地址前缀：192.168.0.0/16
  • 名为 ADSubnet 的子网，其地址前缀：192.168.1.0/24
  • 名为 AppSubnet 的子网，其地址前缀：192.168.2.0/24

任务 1 - 创建地缘组

第一个任务是为虚拟网络创建一个地缘组。

  1. 打开浏览器并转到 https://manage.windowsazure.com/。提示时，使用您的 Windows Azure 凭据登录。在 Windows Azure 门户中，单击“网络”、“地缘组”，然后在底部单击“创建”。

  2. 将该地缘组命名为 adag、包括说明并选择一个区域。单击“完成”按钮创建该地缘组。

    创建地缘组

任务 2 - 新建虚拟网络

下一步是在您的订阅中新建一个虚拟网络。

  1. 在 Windows Azure 门户中，单击“网络”、“虚拟网络”，然后在底部单击“创建”。

  2. 指定虚拟网络名称 ADVNET 和说明，然后单击箭头按钮继续。

    新建虚拟网络

  3. 将“地址空间”值设置为 192.168.0.0/16 并添加 2 个子网，一个名为 ADSubnet（前缀为 192.168.1.0/24），另一个名为 AppSubnet（前缀为 192.168.2.0/24）。

    添加地址空间和子网

  4. 保留 DNS 的默认设置并单击完成按钮。

    创建虚拟网络

练习 2：部署第一个域控制器

我们可以选择是使用 Windows Azure 门户还是 PowerShell 来设置将成为域控制器的虚拟机。在此实验中，您将使用 Windows Azure 门户。

任务 1 - 新建虚拟机

  1. 在 Windows Azure 门户中，单击“新建”，然后选择“虚拟机”、“从库中”。

    从库中新建虚拟机

  2. 单击左侧菜单中的“平台映像”，然后从列表中选择 Windows Server 2008 R2 SP1 映像。单击箭头继续。

  3. 输入 ad-dc 作为“虚拟机名称”并设置 Administrator 用户的密码。单击箭头继续。

    新建虚拟机

  4. DNS 名称需要是唯一的，以使此实验有创意。您可以选择某一现有存储帐户，或者自动生成一个新的存储帐户。从下拉列表“区域/地缘组/虚拟网络”中选择您的虚拟网络。单击箭头继续。

    为虚拟机选择虚拟网络

  5. 从“虚拟网络子网”列表中选中 ADSUBNET 选项。单击“完成”按钮创建虚拟机。

    选择 AD 子网

  6. 请等待虚拟机创建完成。单击该虚拟机名称并选择“仪表板”。

  7. 单击页面底部菜单中的“附加”，然后选择“附加空磁盘”。

    附加空磁盘

  8. 在“将空磁盘附加到虚拟机”页中，将“大小”设置为 20 GB 并单击“完成”按钮以创建磁盘。

    创建新磁盘

任务 2 - 创建域控制器

  1. 在前一任务中创建的虚拟机状态变为“正在运行”后，选择它并单击“连接”。如果您使用了建议的名称，此虚拟机应名为 ad-dc。

  2. 系统会要求您下载远程桌面客户端。单击“打开”并使用您在创建虚拟机时定义的管理员凭据登录。

  3. 在 Azure 虚拟机中，启动控制台会话并运行命令 IPConfig，然后记录返回的 IPv4 IP 地址。

  4. 通过转到“开始”|“所有程序”|“管理工具”|“计算机管理”，打开“计算机管理”。

  5. 展开“存储”节点，然后选择“磁盘管理”。

    打开磁盘管理

  6. 通过单击“初始化磁盘”对话框中的“确定”，初始化磁盘 2。

    初始化磁盘

  7. 右键单击未分配的磁盘空间，选择“新建简单卷”。

    创建新简单卷

  8. 将启动“新建简单卷向导”。单击“下一步”并接受“指定卷大小”和“分配驱动器号或路径”屏幕中的默认值。

  9. 在“格式化分区”屏幕中，将“卷标”设置为 DIT，然后单击“下一步”。

  10. 单击“完成”以开始格式化。格式化完成后，磁盘就可以使用了。

    已格式化的磁盘

  11. 现在，将启动 Active Directory 域服务安装向导。为此，请单击“开始”和“运行”，然后键入 DCPromo 并按 Enter。等待 ADDS 二进制文件安装完成。

    Active Directory 域服务安装向导

  12. 单击两次“下一步”。

  13. 选择“在新林中新建域”，单击“下一步”。

    在新林中新建域

  14. 将“目录林根级域”命名为 contoso.com，单击“下一步”。

    设置域名

  15. 将功能级别设置为 Windows Server 2008 R2。单击“下一步”，等待该过程完成。

    选择林功能级别

  16. 使用默认选择创建 DNS 服务器。单击“下一步”。

    选择其他选项

  17. 选择“是”，计算机将使用由 DHCP 服务器自动分配的 IP 地址（不推荐）。

    使用自动分配的 IP

    注意：使用带有 Windows Azure IaaS 的虚拟网络时，IP 地址的生存期为虚拟机租赁期。不要将 IP 地址设置为静态。

  18. 在此实验中不集成到现有的 AD 环境中，因此单击“是”。

    DNS 创建警告

  19. 将数据库、日志文件和 SYSVOL 文件夹位置设置为刚刚格式化的数据磁盘（例如 F:\NTDS 用于数据库，F:\NTDSLogs 用于日志，F:\SYSVOL 用于 SYSVOL）。单击“下一步”继续。

    为数据库、日志和 SYSVOL 文件夹设置值

  20. 键入设置原始计算机时所使用的密码，单击“下一步”。

    设置域管理员密码

  21. 最后，再次单击“下一步”并允许对 Active Directory 进行配置，这将需要几分钟。在提示是否允许重新启动时，选择“立即重新启动”。

练习 3：将新虚拟机设置为加入域

下面的示例演示如何自动将新虚拟机设置为在启动时加入 Active Directory 域。

注意：对于此练习，您将需要在上一练习中创建的虚拟机的 IP 地址。

  1. 从“开始”|“所有程序”|Windows Azure|Windows Azure PowerShell 打开 Windows Azure PowerShell，右键单击 Windows Azure Powershell 并选择“以管理员身份运行”。

  2. 执行以下命令以获取可用操作系统磁盘映像的名称。记录 Windows Server 2008 映像磁盘名称。

    PowerShell
    Get-AzureVMImage | Select ImageName

  3. 执行以下命令为新虚拟机定义操作系统磁盘映像名称。

    PowerShell
    $imgName = '[OS-IMAGE-NAME]'

  4. 设置虚拟机的 DNS 设置。为此，您将使用在练习 1 中创建的虚拟机，在其中您配置了 Active Directory。替换占位符，然后执行以下命令。使用在练习开头记录的 IP 地址。

    PowerShell
    $advmIP = '[AD-IP-ADDRESS]'
$advmName = '[AD-VM-NAME]'
# Point to IP Address of Domain Controller Created Earlier
$dns1 = New-AzureDns -Name $advmName -IPAddress $advmIP

  5. 设置新虚拟机的配置设置以在设置过程中自动加入域。将占位符替换为管理员和域密码，然后执行命令。

    PowerShell
    $vmName = 'advm1'
$adminPassword = '[YOUR-PASSWORD]'
$domainPassword = '[YOUR-PASSWORD]'
$domainUser = 'administrator'
$FQDomainName = 'contoso.com'
$subNet = 'AppSubnet'
# Configuring VM to Automatically Join Domain
$advm1 = New-AzureVMConfig -Name $vmName -InstanceSize Small -ImageName $imgName | 
            Add-AzureProvisioningConfig -WindowsDomain -Password $adminPassword `
            -Domain 'contoso' -DomainPassword $domainPassword `
            -DomainUserName $domainUser -JoinDomain $FQDomainName |
     Set-AzureSubnet -SubnetNames $subNet

    注意：前面的命令假定您使用了在此实验中显示的域名和子网的建议名称。如果您使用了不同的名称，可能需要更新这些值。

  6. 使用在前面步骤中定义的域和 DNS 设置来创建新虚拟机。将占位符替换为唯一的服务名称。

    PowerShell
    $serviceName = '[YOUR-SERVICE-NAME]' $affinityGroup = 'adag' $adVNET = 'ADVNET' # New Cloud Service with VNET and DNS settings New-AzureVM -ServiceName $serviceName -AffinityGroup $affinityGroup ` -VMs $advm1 -DnsSettings $dns1 -VNetName $adVNET

    得到的体系结构

  7. 等待设置好虚拟机。

  8. 在 Windows Azure 门户中，通过选择这个新虚拟机并单击底部的“连接”登录到它。

  9. 登录后，在“初始配置任务”窗口中，通过选中“计算机全名”和“域”属性来验证该机已自动加入 contoso.com 域。

    初始配置任务

小结

在此实验中，您演练了使用 Windows Azure 虚拟机和简单的虚拟网络来部署新的 Active Directory 域所需的步骤。
